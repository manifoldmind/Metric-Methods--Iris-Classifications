## #CUTTING : 1. Что значит «разделить на train и test» — огранка

Представь, что ты преподаватель в школе сомелье. У тебя есть 178 бутылок вина, и для каждой ты точно знаешь сорт (это твой датасет). Ты хочешь обучить ученика (модель kNN) определять сорт по анализу.

Но как проверить, что ученик действительно научился, а не просто запомнил ответы для этих 178 бутылок?

**Ты делаешь так:**
- Отдаёшь ученику **124 бутылки** (train) — показываешь их анализ и говоришь сорт. Он тренируется, смотрит на примеры.
- А **54 бутылки** (test) прячешь. Потом даёшь ему их анализы, но без ответов — и просишь угадать сорт.
- Затем ты сверяешь его ответы с настоящими сортами (это и есть оценка).

>[!warning] Если ученик угадал 50 из 54 — точность около 93%. Это значит, что он **обобщил** знания, а не вызубрил.
Вот это и есть **hold-out** — один из способов честной оценки.

---

## 2. Почему в ТЗ есть hold-out и кросс-валидация (связь)

Методичка просит **сравнить два способа оценки**:
- **Hold-out** (один раз случайно разбили, обучили, проверили)
- **Кросс-валидация** (разбили на 5 частей, 5 раз повторили обучение/проверку)

Почему? Потому что hold-out может быть случайно неудачным: если в test попали редкие или «сложные» образцы, оценка занизится. Кросс-валидация даёт более стабильную оценку.

Но сейчас — сначала **hold-out**. Это база, с которой всё начинается.

```python
# Task 1: Hold-out разбиение Wine на train/test

from sklearn.model_selection import train_test_split
import pandas as pd

# Загружаем датасет (если ещё не загружен)
from sklearn.datasets import load_wine
wine = load_wine()

# Превращаем признаки в DataFrame для удобства (можно и без этого)
X = pd.DataFrame(wine.data, columns=wine.feature_names)
y = wine.target

# #NOTE: X — это таблица с 13 столбцами-признаками, y — массив с числами 0,1,2 (классы)

# Разбиваем. random_state — фиксируем для воспроизводимости (каждый раз одинаковое разбиение)
# test_size=0.30 означает 30% данных уходят в test (54 образца из 178)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, 
    test_size=0.30, 
    random_state=42,          # 42 — просто классическое число, можно любое
    stratify=y                # сохраняем пропорцию классов (важно!)
)

# #NOTE: stratify=y — это умный параметр. Он делает так, что в train и test 
# распределение классов будет таким же, как в исходном наборе.
# Без stratify могло бы случиться, например, что в test попадут все образцы класса 2 — тогда оценка будет нечестной.

print(f"Размер train: {X_train.shape[0]} образцов")
print(f"Размер test:  {X_test.shape[0]} образцов")
print("\nРаспределение классов в train:")
print(y_train.value_counts().sort_index())
print("\nРаспределение классов в test:")
print(y_test.value_counts().sort_index())
```

**После запуска ты увидишь:**  
Train: 124 образца (в каждом классе примерно 70% от исходного числа — 41, 50, 34 — что-то похожее)  
Test: 54 образца (соответственно 18, 21, 14)
