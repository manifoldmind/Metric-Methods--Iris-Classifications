## 2. Разбиение данных и метод hold‑out (Task 1)

### #CHILLOUT: почему просто обучить на всех данных — плохо

Допустим, мы учим модель на всех 178 образцах и на них же проверяем. Она может просто запомнить ответы, но не научиться обобщать. Чтобы оценить реальную способность предсказывать новые вина, мы прячем часть образцов (test) и используем их как «экзамен». Остальные (train) — тренировочный набор. Это и есть **hold‑out** — одиночное случайное разбиение. Позже мы увидим, что такой подход может зависеть от удачности разбиения, но для начала он идеален.

#CUTTING: представьте, что вы преподаватель в школе сомелье. Вы даёте студенту 124 карточки с описанием вин и их сортами для тренировки, а 54 карточки прячете. Потом показываете ему спрятанные описания и просите угадать сорт — так вы проверяете, чему он научился.

### #MAKED: выполнение разбиения

```python
from sklearn.model_selection import train_test_split

X = df.drop('target', axis=1)
y = df['target']

X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.30,        # 30% в test → 54 образца
    random_state=42,       # для воспроизводимости
    stratify=y             # сохраняем пропорции классов
)

print(f"Train: {X_train.shape[0]} образцов, Test: {X_test.shape[0]} образцов")
print("\nРаспределение классов в train:")
print(y_train.value_counts().sort_index())
print("\nРаспределение классов в test:")
print(y_test.value_counts().sort_index())
```

#NOTE: `stratify=y` гарантирует, что в каждом наборе сохранится исходное соотношение классов. Без него могло бы случайно оказаться, что в test почти нет вин какого-то сорта — и оценка была бы нечестной.

## 3. Обучение kNN и подбор K на hold‑out (Task 2)

### #CHILLOUT: почему перебираем K от 1 до 15

Параметр K — число ближайших соседей, по которым модель принимает решение о классе.
- Слишком маленькое K (1–2) делает модель чувствительной к шуму.
- Слишком большое K (больше ~20% от обучающей выборки) приводит к недообучению: модель начинает предсказывать самый частый класс, игнорируя тонкие различия.

У нас 124 тренировочных образца. Разумный потолок — около 15–20 соседей. Расширение диапазона, как правило, не улучшает точность, а лишь удлиняет график пустым хвостом. Поэтому мы анализируем K = 1…15 и ищем пик точности на test.

#DEEP: о компромиссе bias-variance при выборе K см. `DEEP — Теория по кросс-валидации.md`.

### #MAKED: цикл по K и построение графика

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score
import matplotlib.pyplot as plt
import numpy as np

k_range = range(1, 16)
train_acc = []
test_acc = []

print(" K | Train acc | Test acc")
print("---|-----------|---------")
for k in k_range:
    knn = KNeighborsClassifier(n_neighbors=k)
    knn.fit(X_train, y_train)
    train_acc.append(accuracy_score(y_train, knn.predict(X_train)))
    test_acc.append(accuracy_score(y_test, knn.predict(X_test)))
    print(f"{k:2d} | {train_acc[-1]:.3f}     | {test_acc[-1]:.3f}")

best_k = k_range[np.argmax(test_acc)]
best_test_acc = max(test_acc)
print(f"\n🏆 Лучшее K = {best_k}, точность на test = {best_test_acc:.3f}")

# График
plt.figure(figsize=(8,5))
plt.plot(k_range, train_acc, 'o--', label='Train accuracy')
plt.plot(k_range, test_acc, 's-', label='Test accuracy')
plt.axvline(x=best_k, color='red', linestyle=':', label=f'Best K = {best_k}')
plt.xlabel('K (число соседей)')
plt.ylabel('Accuracy')
plt.title('kNN на Wine: подбор K по hold‑out')
plt.xticks(k_range)
plt.legend()
plt.grid(True)
plt.show()
```

#NOTE: `np.argmax(test_acc)` возвращает индекс максимального значения в массиве, по нему мы находим оптимальное K.

### #ANALYZED: результат

Оптимальным оказалось **K = 7** с точностью на тестовой выборке около 78–82% (точные значения зависят от разбиения). Это хороший результат для такого простого метода. Модель достаточно гибкая, чтобы улавливать структуру каждого сорта, и при этом не переобучается.

### #CHILLOUT: интерпретация

K=7 означает, что для нового образца мы ищем 7 самых похожих вин из тренировочной коллекции. Если среди них больше всего представителей сорта 0, предсказываем сорт 0, и так далее. Число 7 — компромисс: оно достаточно велико, чтобы единичный «странный» сосед не испортил прогноз, но не настолько велико, чтобы стереть границы между сортами.

#ANS: Да, hold‑out чувствителен к случайному разбиению. При другом `random_state` оптимальным может оказаться K=5 или K=9.

Однако мы получили этот результат только для одного конкретного разбиения. Возможно, с другим `random_state` лучшим оказался бы K=5 или K=9. Чтобы оценить устойчивость выбора, переходим к кросс-валидации.

## 4. Кросс‑валидация: стабильная оценка и выбор K (Task 3)

### #CHILLOUT: идея метода

Одиночный hold‑out похож на экзамен по одному-единственному билету. Можно случайно вытянуть «лёгкие» вопросы и получить завышенную оценку. Кросс‑валидация (CV) решает эту проблему, устраивая 5 экзаменов по разным билетам.

Мы делим все 178 вин на 5 примерно равных частей (фолдов). Затем по очереди:
- Один фолд объявляем тестовым,
- Остальные четыре — тренировочными.
Обучаем модель и измеряем точность на отложенном фолде. В итоге у нас 5 оценок, которые мы усредняем. Так каждая проба побывает и в train, и в test, а итоговая метрика становится гораздо стабильнее.

#CUTTING: представьте 5 экзаменов по разным билетам. Средний балл надёжнее отражает знания, чем оценка за один случайный билет.

#DEEP: формальное определение k‑fold CV и bias‑variance trade‑off см. в `DEEP — Теория по кросс-валидации.md`.

### #MAKED: подбор K с помощью 5‑fold кросс‑валидации

Для учебного сравнения применим кросс‑валидацию **ко всему датасету** (178 образцов). Это покажет, какое K выбирает CV, и позволит напрямую сопоставить с результатом hold‑out.

```python
from sklearn.model_selection import cross_val_score

cv_scores_mean = []
cv_scores_std = []

for k in k_range:
    knn = KNeighborsClassifier(n_neighbors=k)
    # 5-fold CV, метрика accuracy
    scores = cross_val_score(knn, X, y, cv=5, scoring='accuracy')
    cv_scores_mean.append(scores.mean())
    cv_scores_std.append(scores.std())
    print(f"K={k:2d}: средняя CV accuracy = {scores.mean():.3f} ± {scores.std():.3f}")

best_k_cv = k_range[np.argmax(cv_scores_mean)]
print(f"\n🏆 Лучшее K по CV = {best_k_cv}")

# График средних accuracy с разбросом
plt.figure(figsize=(8,5))
plt.errorbar(k_range, cv_scores_mean, yerr=cv_scores_std, 
             fmt='o-', capsize=4, label='CV accuracy (mean ± std)')
plt.axvline(x=best_k_cv, color='red', linestyle=':', label=f'Best K={best_k_cv}')
plt.xlabel('K')
plt.ylabel('Cross‑validated accuracy')
plt.title('5‑fold кросс‑валидация: выбор K')
plt.xticks(k_range)
plt.legend()
plt.grid(True)
plt.show()
```

#NOTE: `cross_val_score` по умолчанию использует `StratifiedKFold` для задач классификации, что сохраняет баланс классов в каждом фолде.

### #ANALYZED: анализ и сравнение

- **Hold‑out** дал K = 7 на одном фиксированном разбиении.
- **CV** часто указывает на K = 5 или K = 7, в зависимости от случайности разбиения на фолды. Небольшое расхождение абсолютно нормально.

Это расхождение — не ошибка, а наглядная иллюстрация того, почему CV предпочтительнее:
- Hold‑out чувствителен к единственному разбиению. При другом `random_state` результат мог быть иным.
- CV усредняет результаты по нескольким разбиениям, сглаживая случайные выбросы. Выбранное на основе CV K будет демонстрировать стабильную точность на новых данных.

Если в лабораторной требуется финальная независимая оценка, то после выбора K с помощью CV мы должны обучить модель с этим K на **всём тренировочном наборе** (или на всём датасете, кроме заранее отложенного test) и протестировать её на тех данных, которые ни разу не участвовали в подборе гиперпараметра.
